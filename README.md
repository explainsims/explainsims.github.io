## **ExplAIn Sims: Interactive Learning, Zero Friction**

Stop wrestling with installation permissions and outdated software. **ExplAIn Sims** is a curated suite of browser-native physics simulations, utility tools, and educational mini-games built for the modern classroom. 

Whether you’re a student diving into a concept or a teacher looking for a seamless demonstration tool, ExplAIn Sims puts interactive science directly in your hands—no admin rights required.

---

### **The ExplAIn Edge**

Why settle for static textbooks when you can simulate? This project was born to solve the "blocked software" headache in schools by prioritizing accessibility and resilience.

* **Browser-First:** No downloads, no installs. If you have a web browser, you’re ready.
* **Active Engagement:** Every tool is built for hands-on experimentation, not just observation.
* **Classroom Ready:** Purpose-built for curriculum revision, practical demonstrations, and lab work.
* **Always Available:** Once you've visited a tool, it caches to your device. You can keep experimenting even when the Wi-Fi cuts out.

> **Pro-Tip:** While most of the library is offline-capable, a few select features require an active internet connection and may require you to use your own API keys.

---

### **Built Lean & Fast**

We believe that educational tech shouldn't be bloated. ExplAIn Sims is built on a "Vanilla" stack to ensure maximum speed and compatibility:

* **Core:** HTML5, CSS3, and Vanilla JavaScript.
* **Architecture:** No heavy frameworks, no complex bundlers, and no build steps.
* **Persistence:** Service Workers handle the heavy lifting for offline caching.
* **Infrastructure:** Hosted via GitHub Pages.

### **Join the Mission**

Have an idea for a new sim or a way to make an existing tool smoother? We love contributions that align with our core philosophy.

#### **Our Design Pillars**
* **Radical Simplicity:** Keep code readable and dependencies to a minimum.
* **Classroom Resilience:** UI must be mobile-friendly, touch-responsive, and low-friction.
* **Independence:** Every HTML page should function as a standalone application.

#### **Dev Notes**
* This is a **static site**. Modifications happen directly within the repository.
* **Crucial:** If you update pre-cached files, remember to increment the `BUILD_ID` in `sw.js` to trigger a refresh for our users.
